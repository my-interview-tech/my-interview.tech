Самый распространенный тип связи. Один элемент из первой таблицы может быть связан с несколькими элементами из второй таблицы.

Пример: Авторы и Книги

```sql
-- Создание таблицы авторов
CREATE TABLE authors (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100)
);

-- Создание таблицы книг с внешним ключом на автора
CREATE TABLE books (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    isbn VARCHAR(13),
    author_id INT,
    FOREIGN KEY (author_id) REFERENCES authors(id)
);

-- Вставка данных
INSERT INTO authors (name, email) VALUES
('Джоан Роулинг', 'joan@example.com'),
('Джордж Мартин', 'george@example.com');

INSERT INTO books (title, isbn, author_id) VALUES
('Гарри Поттер и философский камень', '978-5-17-087885-4', 1),
('Гарри Поттер и тайная комната', '978-5-17-087886-1', 1),
('Игра престолов', '978-5-17-087887-8', 2),
('Битва королей', '978-5-17-087888-5', 2);
```
