3. Безопасность cookie и новые стандарты
- **Privacy Sandbox**
  - Инициатива по отказу от сторонних cookie
  - Повышение безопасности веб-адресов
  - API Trust Token как альтернативный механизм

- **Угрозы безопасности для cookie:**
  - Межсайтовый скриптинг (XSS) - внедрение произвольного JS-кода
  - Сниффинг - перехват cookie
  - Хранение конфиденциальной информации в cookie (риск)
  - Взлом сессии через получение ID сессии из cookie

- **Защита cookie:**
  - Использование httpOnly атрибута
  - Применение SSL/TLS + secure атрибута
  - Ограничение через domain и path
  - Установка срока жизни через max-age
  - Возможность установки cookie как на клиенте, так и на сервере

- **Дополнительные механизмы безопасности:**
  - Ограничение попыток входа на сайт

___

4. Системы защиты веб-приложений
- **WAF (Web Application Firewall)**
  - Брандмауэр веб-приложений для защиты на прикладном уровне
  - Защита от XSS и SQL-инъекций
- **Обход WAF**
  - Существуют методы обхода защиты WAF
- **RASP (Runtime Application Self-Protection)**
  - Защита приложения во время выполнения

___
