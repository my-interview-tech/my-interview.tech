Одной из серьезных угроз для JavaScript-разработчиков являются атаки через компрометацию npm пакетов. Злоумышленники могут:

- Публиковать пакеты со схожими названиями популярных библиотек (typosquatting)
- Перехватывать контроль над существующими пакетами
- Внедрять вредоносный код в зависимости популярных пакетов

**Примеры реальных атак:**

- Пакет `event-stream` (2018) – внедрение бэкдора для кражи Bitcoin-кошельков
- `ua-parser-js` (2021) – добавление майнеров криптовалюты и троянов для кражи паролей
- `colors` и `faker` (2022) – преднамеренное повреждение пакетов автором

**Меры предосторожности:**

- Используйте `npm ci` вместо `npm install` в производственной среде
- Фиксируйте версии зависимостей в `package-lock.json`
- Применяйте инструменты аудита: `npm audit`, Snyk, SonarQube
- Рассмотрите использование приватных репозиториев для критически важных зависимостей
