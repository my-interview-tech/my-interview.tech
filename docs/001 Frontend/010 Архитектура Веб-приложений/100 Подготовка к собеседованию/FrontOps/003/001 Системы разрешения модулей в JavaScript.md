**Resolver** — это алгоритм, который находит по использованию символа в коде его декларацию. Другими словами, это механизм, определяющий как именно интерпретировать такие инструкции как `require('./a')` или `import { Component } from 'react'`.

В чем сложность задачи?

Модуль может быть взят из различных источников:
- Относительный путь (`./utils.js`)
- Абсолютный путь (`/src/components/Button.js`)
- Путь с алиасом (`@components/Button`)
- Из node_modules (безымянный импорт `react`)
- Из глобального пространства
- Через символические ссылки (symlinks)

При этом нужно учитывать:

- Встроенные модули Node.js
- Различные расширения файлов (`.js`, `.mjs`, `.cjs`, `.json`, `.ts`, `.tsx`, `.jsx`)
- Специфичные механизмы для конкретных систем сборки
- Возможность работы не только с файловой системой (HTTP, виртуальные модули)
