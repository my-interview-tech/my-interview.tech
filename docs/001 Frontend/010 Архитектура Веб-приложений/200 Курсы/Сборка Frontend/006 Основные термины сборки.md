Минификация и сжатие
В чем разница?
Минификация - процесс удаления ненужных символов из исходного кода без изменения его функциональности

Сжатие - алгоритмическое преобразования данных, производимое с целью уменьшения занимаемого ими объема

Минификацию мы делаем на этапе сборки
Компрессию можно делать и на этапе сборки (это бессмысленно) обычно это делают в рантайме

Хэш
это строка фиксированной длины, которая создается из исходных данных с помощью хеш-функции
В сборке используется для генерации уникального имени чанка
Хэш должен быть устойчивым и не меняться если содержимое чанка не изменилось

Плагин/лоадер

Механизм расширения возможностей работы сборщика
интерфейс написания плагина декларирует каждый сборщик
лоадеры есть только в вебпаке (по сути это тип плагина)

Watch mode
Режим работы сборщика при котором он следит за имзенениями в подключаемых файлах
В случае внесения изменений запускается пересборка
Пересборка вызывается только для измененного файла
