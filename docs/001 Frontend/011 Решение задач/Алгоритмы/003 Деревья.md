**Дерево** — это нелинейная динамическая структура данных, организованная в виде иерархии элементов, называемых узлами (вершинами), между которыми существуют связи, именуемые ветвями.

В зависимости от типа дерева у вершины может быть разное количество дочерних вершин. Это число называется степенью вершины. Его максимальное значение определяет степень дерева в целом. Если у вершины дерева имеется N дочерних вершин, то мы имеем дело с N-арным деревом. При этом следующие за ними дочерние вершины рассматриваются как потомки - это дочерние вершины дочерних вершин. Родительские вершины родительских вершин называются предками. Если вершина не имеет других потомков, то ее называют листом или термиальной вершиной.

```
ЗДЕСЬ ДОЛЖНО БЫТЬ ИЗОБРАЖЕНИЕ
```

Обход дерева осуществляется перебором дочерних вершин и рекурсивным обходом всех поддеревьев, связанных с этими дочерними вершинами (рекурсивный спуск). Поскольку большинство деревьев относительно неглубокие - много детей, но всего несколько уровней иерархии, то это создает предпосылки для написания эффективного кода. Исключение составляют генеалогии человека (много уровней предков  и всего несколько дочерних вершин на уровень).

При необходимости в узлах можно оставлять обратные указатели, которые позволяют обратно подниматься по дереву , но их сложнее программно поддерживать.

Типовым решением задачи обхода дерева является создание двух функций. Одна - для получения адреса корневой вершины, а вторая функция - для рекурсивного спуска по ветвям дерева. Она имеет такие дополнительные параметры, как ссылки на предка и потомка, а также информацию о глубине дерева.
