## Что такое npx?

Утилита **`npx`** позволяет выполнять пакеты из реестра npm без их предварительной установки в систему. Она входит в состав npm начиная с версии 5.2.0 и предоставляет удобный способ использования пакетов одноразово или для тестирования.

## Как выполнить утилиту без её установки

Чтобы выполнить утилиту с помощью **`npx`** без её предварительной установки, достаточно просто вызвать команду через **`npx`**. **`npx`** автоматически загрузит утилиту из **npm** и выполнит её, если она ещё не установлена на вашем компьютере.

### Примеры использования

#### 1. Запуск утилиты, которая не установлена глобально или локально

Допустим, вы хотите использовать утилиту **`cowsay`** (которая выводит текст в виде речи коровы) без её установки. Просто выполните команду через **`npx`**:

```bash
npx cowsay "Hello, world!"
```

В этом случае **`npx`** выполнит следующие шаги:

- Проверит, установлен ли пакет **`cowsay`**
- Если он не найден, **`npx`** автоматически скачает последнюю версию пакета с **npm**, выполнит команду и затем удалит его

#### 2. Запуск одноразовой утилиты (например, для создания приложения)

Если вы хотите создать проект с помощью **`create-react-app`** или другого генератора, который не установлен в системе, можно использовать **`npx`** для его запуска без предварительной установки:

```bash
npx create-react-app my-app
```

Эта команда:

- Автоматически загрузит последнюю версию **`create-react-app`**
- Создаст новый проект **`my-app`** с помощью этого пакета, не устанавливая его глобально

### Запуск конкретной версии пакета

Вы можете также указать конкретную версию пакета, которую хотите запустить:

```bash
npx cowsay@2.0.0 "Использую версию 2.0.0!"
```

### Использование с локальными пакетами

Если пакет уже установлен локально в вашем проекте, **`npx`** сначала будет искать его в локальных `node_modules`, и только потом загружать из реестра npm:

```bash
# Если eslint установлен локально в проекте
npx eslint --init
```

## Преимущества использования npx

- **Безопасность:** Вы не загромождаете вашу систему лишними глобальными пакетами
- **Экономия времени:** Не нужно устанавливать утилиты, которые используются лишь изредка
- **Свежие версии:** При каждом вызове **`npx`** будет загружаться последняя версия пакета с **npm**, что гарантирует использование актуальной версии инструмента
- **Изоляция**: Каждый запуск осуществляется в изолированной среде, что уменьшает конфликты между зависимостями

## Дополнительные возможности npx

### Запуск локальных скриптов из node_modules/.bin

```bash
# Вместо ./node_modules/.bin/jest
npx jest
```

### Выполнение команды из GitHub репозитория

```bash
npx github:user/repo
# или
npx https://github.com/user/repo
```

### Запуск gist с GitHub

```bash
npx gist:gist-id
```

## Заключение

С помощью **`npx`** вы можете легко запускать утилиты и пакеты, не устанавливая их глобально или локально, что упрощает использование одноразовых инструментов и уменьшает количество установленных пакетов в вашем проекте. Это особенно полезно для инструментов, которые быстро развиваются или которые вы используете нечасто.
