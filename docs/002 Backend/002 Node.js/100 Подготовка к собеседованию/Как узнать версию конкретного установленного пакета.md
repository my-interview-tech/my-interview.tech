Чтобы узнать версию конкретного установленного пакета в **Node.js**, можно использовать несколько методов. Рассмотрим основные способы:

## 1. Использование команды `npm list`

Для того чтобы узнать версию установленного пакета, можно использовать команду `npm list`, указав имя пакета.

```bash
npm list <пакет>
```

### Пример для `lodash`:

```bash
npm list lodash
```

Этот вывод покажет точную версию пакета, установленного в вашем проекте:

```
my-project@1.0.0
└── lodash@4.17.21
```

Если вы хотите ограничить вывод только этой версией, можно использовать опцию `--depth=0`:

```bash
npm list lodash --depth=0
```

## 2. Просмотр в `package.json`

Если вы установили пакет как зависимость, его версия будет указана в вашем файле **`package.json`** в секции `dependencies` или `devDependencies`.

```json
"dependencies": {
  "lodash": "^4.17.21"
}
```

## 3. Использование команды `npm show`

Если вы хотите узнать версию пакета, который установлен в вашем проекте, или доступную последнюю версию, используйте команду `npm show`.

```bash
npm show <пакет> version
```

### Пример для `lodash`:

```bash
npm show lodash version
```

Это вернет текущую доступную версию пакета из реестра npm.

## 4. Просмотр директории `node_modules`

Также можно посмотреть версию конкретного пакета, заглянув в файл `package.json` внутри директории пакета в **`node_modules`**:

```bash
cat node_modules/lodash/package.json | grep version
```

Это покажет строку с версией пакета, например:

```
"version": "4.17.21"
```

## 5. Использование `npx` для глобальных пакетов

Если вы хотите узнать версию глобально установленного пакета с CLI-интерфейсом, используйте команду с `npx`:

```bash
npx <пакет> --version
```

### Пример для `typescript`:

```bash
npx typescript --version
```

## 6. Использование Node.js API

В коде JavaScript можно программно получить версию пакета:

```javascript
const pkg = require("пакет/package.json")
console.log(pkg.version) // Выведет версию пакета
```

## Сравнение методов

| Метод                          | Преимущества                               | Ограничения                                          |
| ------------------------------ | ------------------------------------------ | ---------------------------------------------------- |
| `npm list <пакет>`             | Показывает фактически установленную версию | Только для установленных пакетов                     |
| `package.json`                 | Быстрый просмотр                           | Может содержать диапазоны версий, а не точную версию |
| `npm show <пакет> version`     | Показывает актуальную версию               | Требует подключения к интернету                      |
| `node_modules/**/package.json` | Точная информация о пакете                 | Зависит от структуры проекта                         |
| `npx <пакет> --version`        | Работает для CLI-инструментов              | Не для всех пакетов                                  |

## Заключение

- Для быстрой проверки используйте **`npm list <пакет> --depth=0`**
- Для проверки последней доступной версии используйте **`npm show <пакет> version`**
- Для просмотра зависимостей проекта смотрите в **`package.json`**
