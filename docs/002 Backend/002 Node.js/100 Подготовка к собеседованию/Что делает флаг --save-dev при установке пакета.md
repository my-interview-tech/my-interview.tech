Флаг **`--save-dev`** при установке пакета с помощью **npm** используется для добавления пакета как **зависимости для разработки** в проект.

Когда вы используете **`--save-dev`**, npm добавляет установленные пакеты в раздел **`devDependencies`** в файле **`package.json`**. Это означает, что данный пакет необходим только в процессе разработки, но не в производственной (продакшн) среде.

### Когда использовать **`--save-dev`**?

- Для пакетов, которые нужны только на этапе разработки, например:
    - Тестовые фреймворки (например, **Jest**, **Mocha**).
    - Инструменты для сборки проекта (например, **Webpack**, **Babel**).
    - Линтеры и форматеры (например, **ESLint**, **Prettier**).
    - Инструменты для документирования (например, **Swagger**).

Эти пакеты не требуются в продакшн-среде, поэтому они добавляются в **`devDependencies`**, чтобы не загромождать ваш продакшн-код и не увеличивать объём приложения.

### Пример:

Когда вы устанавливаете пакет с флагом **`--save-dev`**, например, **Webpack** для сборки проекта:

`npm install webpack --save-dev`

Это приведет к добавлению **webpack** в раздел **`devDependencies`** в **`package.json`**:

`"devDependencies": {   "webpack": "^5.0.0" }`

### Важные моменты:

- Пакеты, установленные с флагом **`--save-dev`**, не устанавливаются при выполнении **`npm install --production`**, что помогает уменьшить размер продакшн-среды.
- Это полезно для разделения зависимостей, которые нужны для разработки, и тех, которые нужны в продакшн-среде (например, **`express`** для сервера будет в разделе **`dependencies`**).

### Заключение:

**`--save-dev`** помогает организовать зависимости в проекте, добавляя пакеты, которые нужны только в процессе разработки, в отдельный раздел **`devDependencies`**. Это позволяет эффективно управлять зависимостями и уменьшать объём кода в продакшн-среде.
