Чтобы установить пакет как зависимость для разработки (то есть, чтобы он попал в раздел **`devDependencies`** вашего проекта), используйте команду **`npm install`** с флагом **`--save-dev`**.

### Команда для установки пакета как зависимость разработки:

bash

КопироватьРедактировать

`npm install <пакет> --save-dev`

#### Пример:

Если вы хотите установить, например, **`webpack`** как зависимость разработки, выполните команду:

bash

КопироватьРедактировать

`npm install webpack --save-dev`

### Что происходит:

- Пакет будет установлен в папку **`node_modules`** вашего проекта.
- В **`package.json`** будет добавлен раздел **`devDependencies`**, если его нет, или обновлён с новым пакетом.

#### Пример **`package.json`** после установки:

json

КопироватьРедактировать

`"devDependencies": {   "webpack": "^5.0.0" }`

### Примечание:

- Зависимости, указанные в **`devDependencies`**, **не устанавливаются** при развертывании проекта в **продакшн-среде**, если использовать команду **`npm install --production`** или если переменная окружения `NODE_ENV` установлена в значение **`production`**.

Таким образом, флаг **`--save-dev`** используется для установки пакетов, которые необходимы только на этапе разработки (например, для тестирования, сборки, линтинга и т. д.).
