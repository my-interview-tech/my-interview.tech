Чтобы установить пакет как зависимость для разработки (то есть, чтобы он попал в раздел **`devDependencies`** вашего проекта), используйте команду **`npm install`** с флагом **`--save-dev`** или его сокращенным вариантом **`-D`**.

## Базовый синтаксис

```bash
npm install <пакет> --save-dev
```

или сокращенно:

```bash
npm install <пакет> -D
```

## Пример установки

Если вы хотите установить, например, **`webpack`** как зависимость разработки:

```bash
npm install webpack --save-dev
```

## Что происходит при установке

- Пакет будет установлен в папку **`node_modules`** вашего проекта.
- В **`package.json`** будет добавлен раздел **`devDependencies`**, если его нет, или обновлён с новым пакетом.

## Пример обновленного package.json

После выполнения команды из примера выше, в вашем `package.json` появится запись:

```json
{
  "devDependencies": {
    "webpack": "^5.0.0"
  }
}
```

## Несколько пакетов одновременно

Вы можете установить несколько пакетов как dev-зависимости одной командой:

```bash
npm install webpack webpack-cli babel-loader --save-dev
```

## Использование других пакетных менеджеров

### Yarn

```bash
yarn add webpack --dev
```

### pnpm

```bash
pnpm add webpack -D
```

## Важное примечание

Зависимости, указанные в **`devDependencies`**, **не устанавливаются** при развертывании проекта в **продакшн-среде**, если использовать команду **`npm install --production`** или если переменная окружения `NODE_ENV` установлена в значение **`production`**.

## Типичные пакеты для devDependencies

В раздел `devDependencies` обычно помещают:

- Инструменты для тестирования (Jest, Mocha, Chai)
- Линтеры и форматтеры (ESLint, Prettier)
- Сборщики и транспиляторы (Webpack, Babel, TypeScript)
- Инструменты разработки (Nodemon, тестовые фреймворки)
- Пакеты для документации

Это помогает сохранить продакшн-сборку легкой и минимизировать потенциальные уязвимости.
