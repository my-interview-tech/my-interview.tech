**`ws`** и **`wss`** — это два протокола, используемых для WebSocket-соединений. Основное различие между ними заключается в уровне безопасности соединения:

### 1. **`ws` (WebSocket)**:

- **Протокол**: `ws` — это обычное WebSocket-соединение, которое не использует шифрование.
- **Порт**: Обычно используется на порту `80`, так как он является стандартным для HTTP-соединений.
- **Тип соединения**: Не зашифрованное, передает данные в открытом виде.
- **Применение**: Используется в тех случаях, когда безопасность не является приоритетом, например, для внутренних приложений или в сети с гарантированной безопасностью.

Пример подключения с использованием **`ws`**:

javascript

КопироватьРедактировать

`const ws = new WebSocket('ws://example.com');`

### 2. **`wss` (WebSocket Secure)**:

- **Протокол**: `wss` — это безопасный протокол WebSocket, использующий шифрование с помощью SSL/TLS.
- **Порт**: Обычно используется на порту `443`, так как он является стандартом для HTTPS.
- **Тип соединения**: Зашифрованное соединение, данные передаются через защищённый канал.
- **Применение**: Используется в случаях, когда важна безопасность передаваемых данных, например, в публичных веб-приложениях, где требуется защита от атак типа "Man-in-the-Middle" (MITM).

Пример подключения с использованием **`wss`**:

javascript

`const ws = new WebSocket('wss://example.com');`

### Ключевые различия:

|Характеристика|**ws** (WebSocket)|**wss** (WebSocket Secure)|
|---|---|---|
|**Шифрование**|Нет|Да (SSL/TLS)|
|**Безопасность**|Менее безопасно|Более безопасно|
|**Порт**|80 (по умолчанию)|443 (по умолчанию)|
|**Протокол**|`ws://`|`wss://`|

### Когда использовать:

- **`ws`**: Может использоваться для локальных приложений или когда передаваемые данные не требуют шифрования.
- **`wss`**: Рекомендуется использовать для всех публичных веб-приложений, особенно если важно обеспечить безопасность соединений и передаваемых данных (например, для аутентификации, конфиденциальных данных и т.д.).

Таким образом, основное отличие заключается в уровне безопасности: **`ws`** не использует шифрование, а **`wss`** использует защищённое соединение через SSL/TLS.
