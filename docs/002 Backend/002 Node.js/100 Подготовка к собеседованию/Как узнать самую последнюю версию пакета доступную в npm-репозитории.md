Чтобы узнать самую последнюю доступную версию пакета в **npm-репозитории**, можно использовать несколько команд:

### 1. **Использование команды `npm show`**

Команда **`npm show`** позволяет получить информацию о пакете, включая последнюю доступную версию.

#### Пример:

bash

КопироватьРедактировать

`npm show <пакет> version`

**Пример для пакета `lodash`:**

bash

КопироватьРедактировать

`npm show lodash version`

Это вернет самую последнюю стабильную версию пакета в реестре npm:

bash

КопироватьРедактировать

`4.17.21`

### 2. **Использование команды `npm view`**

Команда **`npm view`** (аналогична `npm show`) также позволяет получить информацию о пакете, включая его версию.

#### Пример:

bash

КопироватьРедактировать

`npm view <пакет> version`

**Пример для пакета `express`:**

bash

КопироватьРедактировать

`npm view express version`

Это также вернет последнюю доступную версию пакета:

bash

КопироватьРедактировать

`4.17.1`

### 3. **Использование веб-страницы npm**

Вы можете всегда посетить страницу пакета на [npmjs.com](https://www.npmjs.com/) и найти самую последнюю версию пакета, которая отображается в верхней части страницы пакета.

Пример для `lodash`:

- Перейдите по адресу [https://www.npmjs.com/package/lodash](https://www.npmjs.com/package/lodash), чтобы увидеть последнюю версию пакета.

### 4. **Использование команды `npm outdated`**

Команда **`npm outdated`** показывает устаревшие пакеты в вашем проекте и их последние доступные версии.

#### Пример:

bash

КопироватьРедактировать

`npm outdated`

Это покажет текущие версии пакетов в вашем проекте и последние доступные версии для обновления.

### Заключение:

- **`npm show <пакет> version`** или **`npm view <пакет> version`** — самые прямые способы узнать последнюю версию пакета.
- **`npm outdated`** — показывает устаревшие пакеты и их последние версии для текущего проекта.
- **npmjs.com** — можно вручную проверить версию пакета на сайте npm.
