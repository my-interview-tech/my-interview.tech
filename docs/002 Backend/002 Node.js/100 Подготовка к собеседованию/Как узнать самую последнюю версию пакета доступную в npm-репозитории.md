Чтобы узнать самую последнюю доступную версию пакета в **npm-репозитории**, можно использовать несколько команд:

## 1. Использование команды `npm show` / `npm view`

Команда **`npm show`** (или ее синоним **`npm view`**) позволяет получить информацию о пакете, включая последнюю доступную версию.

```bash
npm show <пакет> version
```

### Пример для пакета `lodash`:

```bash
npm show lodash version
```

Это вернет самую последнюю стабильную версию пакета в реестре npm:

```
4.17.21
```

## 2. Получение полной информации о пакете

Чтобы получить более детальную информацию о пакете, включая все доступные версии:

```bash
npm view <пакет> versions
```

### Пример:

```bash
npm view express versions
```

Это выведет массив всех когда-либо опубликованных версий пакета:

```
[
  '0.0.1', '0.0.2', ..., '4.17.1', '4.17.2', '4.18.1', '4.18.2'
]
```

## 3. Использование команды `npm outdated`

Команда **`npm outdated`** показывает устаревшие пакеты в вашем проекте и их последние доступные версии.

```bash
npm outdated
```

### Пример вывода:

```
Package   Current  Wanted  Latest  Location
express    4.17.1  4.17.3  4.18.2  project
lodash    4.17.20  4.17.21  4.17.21  project
```

В столбце "Latest" указаны самые последние версии пакетов.

## 4. Использование веб-интерфейса npm

Вы можете всегда посетить страницу пакета на [npmjs.com](https://www.npmjs.com/) и найти самую последнюю версию пакета, которая отображается в верхней части страницы.

### Например:

- Для пакета `lodash`: https://www.npmjs.com/package/lodash
- Для пакета `express`: https://www.npmjs.com/package/express

## 5. Использование утилиты `npm-check-updates`

Для масштабной проверки обновлений можно использовать специальную утилиту:

```bash
# Установка утилиты
npm install -g npm-check-updates

# Проверка доступных обновлений в проекте
ncu

# Обновление всех пакетов до последних версий
ncu -u
```

## 6. Использование команды `npm dist-tag`

Чтобы увидеть последнюю версию пакета с определенными тегами (например, `latest`, `next`, `beta`):

```bash
npm dist-tag ls <пакет>
```

### Пример:

```bash
npm dist-tag ls react
```

Может вернуть что-то вроде:

```
latest: 18.2.0
next: 19.0.0-alpha.1
experimental: 0.0.0-experimental-1234
```

## Сравнение подходов

| Метод                       | Преимущества                                    | Ограничения                   |
| --------------------------- | ----------------------------------------------- | ----------------------------- |
| `npm show <пакет> version`  | Быстрый, точный результат                       | Показывает только одну версию |
| `npm view <пакет> versions` | Показывает историю версий                       | Может быть слишком объемным   |
| `npm outdated`              | Сравнивает с текущими пакетами                  | Требует существующий проект   |
| npmjs.com                   | Визуальный интерфейс, дополнительная информация | Требует браузер               |
| `npm-check-updates`         | Массовая проверка и обновление                  | Требует отдельной установки   |

## Заключение

- Самый быстрый способ: **`npm show <пакет> version`**
- Для сравнения с установленными пакетами: **`npm outdated`**
- Для визуального просмотра и дополнительной информации: **веб-сайт npm**
