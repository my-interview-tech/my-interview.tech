Чтобы с помощью **`npx`** запустить код с разными версиями **Node.js**, можно использовать утилиту **`npx`** совместно с пакетом **`node`** в комбинации с **`n`** или **`nvm`** (если они установлены).

1. **Использование `npx` с пакетом `node`**:
    
    Вы можете указать конкретную версию **Node.js** для выполнения кода через **`npx`**, используя команду **`npx`** с флагом для конкретной версии **Node.js**. Однако, это не всегда так просто, так как **`npx`** в основном управляет пакетами, а не версиями Node.js напрямую.
    
2. **Использование `npx` с менеджерами версий Node.js**:
    
    Для того чтобы запускать код с различными версиями **Node.js**, удобнее всего использовать **менеджеры версий** **Node.js**, такие как **`nvm` (Node Version Manager)** или **`n`**, которые позволяют переключать версии Node.js на лету.
    

### Как использовать `npx` с `n` (Node Version Manager):

**`n`** — это простой менеджер для управления версиями **Node.js**, который позволяет вам быстро переключаться между версиями **Node.js**.

#### Шаги:

1. Установите **`n`** (если ещё не установлен):
    
    Сначала установите **`n`**, используя **npm**:
    
    bash
    
    КопироватьРедактировать
    
    `npm install -g n`
    
2. Используйте **`n`** для выбора нужной версии Node.js:
    
    После того как **`n`** установлен, вы можете переключиться на нужную версию **Node.js**:
    
    bash
    
    КопироватьРедактировать
    
    `n <версия>`
    
    Например:
    
    bash
    
    КопироватьРедактировать
    
    `n 14.17.0`
    
3. После переключения на нужную версию, используйте **`npx`** для запуска кода с этой версией **Node.js**.
    
    Например, если вы хотите запустить утилиту с конкретной версией **Node.js**, сначала переключитесь на нужную версию с помощью **`n`**, а затем выполните команду с **`npx`**:
    
    bash
    
    КопироватьРедактировать
    
    `npx <пакет>`
    

### Как использовать `npx` с `nvm` (Node Version Manager):

**`nvm`** — это ещё один популярный менеджер версий **Node.js**, который позволяет легко переключаться между версиями **Node.js**.

#### Шаги:

4. Установите **`nvm`** (если ещё не установлен):
    
    Для установки **`nvm`** следуйте инструкциям в официальном репозитории: [https://github.com/nvm-sh/nvm](https://github.com/nvm-sh/nvm)
    
5. Установите нужную версию **Node.js** через **`nvm`**:
    
    bash
    
    КопироватьРедактировать
    
    `nvm install <версия>`
    
    Например:
    
    bash
    
    КопироватьРедактировать
    
    `nvm install 14.17.0`
    
6. Переключитесь на нужную версию **Node.js**:
    
    bash
    
    КопироватьРедактировать
    
    `nvm use <версия>`
    
    Например:
    
    bash
    
    КопироватьРедактировать
    
    `nvm use 14.17.0`
    
7. Теперь, когда вы используете нужную версию, можно запускать **`npx`** с этой версией **Node.js**:
    
    bash
    
    КопироватьРедактировать
    
    `npx <пакет>`
    

### Пример использования:

Если вы хотите выполнить код с **Node.js 14.17.0**, то:

8. Установите и переключитесь на нужную версию с помощью **`nvm`**:
    
    bash
    
    КопироватьРедактировать
    
    `nvm install 14.17.0 nvm use 14.17.0`
    
9. Запустите команду с **`npx`**:
    
    bash
    
    КопироватьРедактировать
    
    `npx cowsay "Hello, world!"`
    

Таким образом, используя **`nvm`** или **`n`**, вы можете запускать код с разными версиями **Node.js** и использовать **`npx`** для выполнения команд с любой нужной версией.

### Заключение:

- **`npx`** не управляет версиями **Node.js**, но можно использовать его в связке с инструментами, такими как **`nvm`** или **`n`**, для переключения версий Node.js и запуска команд с нужной версией.
- **`nvm`** и **`n`** — это менеджеры версий Node.js, которые позволяют легко переключаться между версиями для выполнения кода с конкретной версией.
