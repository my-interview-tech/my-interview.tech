Для установки конкретной версии пакета в Node.js проекте можно использовать несколько подходов. Рассмотрим их на примере установки React версии 16.8.1.

## Основной способ — указание версии через @

Самый распространенный способ установить конкретную версию пакета — использовать символ `@` после имени пакета:

```bash
npm install react@16.8.1
```

Эта команда установит именно версию 16.8.1 библиотеки React и добавит её в `dependencies` в вашем файле `package.json`.

## Установка как зависимость для разработки

Если нужно установить конкретную версию как dev-зависимость:

```bash
npm install react@16.8.1 --save-dev
```

## Установка с точной версией

Чтобы установить пакет с точной версией без знаков ^ или ~ в package.json:

```bash
npm install react@16.8.1 --save-exact
```

При использовании флага `--save-exact` в package.json будет записана точная версия `"react": "16.8.1"` вместо `"react": "^16.8.1"`.

## Пакеты с тегами

Некоторые пакеты имеют теги версий (например, `latest`, `next`, `beta`):

```bash
npm install react@latest
npm install react@next
npm install react@beta
```

## Обновление нескольких пакетов до конкретной версии

```bash
npm install react@16.8.1 react-dom@16.8.1 react-router@5.0.0
```

## Использование package.json

Можно также напрямую отредактировать `package.json` и затем выполнить установку:

```json
{
  "dependencies": {
    "react": "16.8.1"
  }
}
```

После этого выполнить:

```bash
npm install
```

## Использование других пакетных менеджеров

### Yarn

```bash
yarn add react@16.8.1
yarn add react@16.8.1 --exact  # для точной версии
```

### pnpm

```bash
pnpm add react@16.8.1
pnpm add react@16.8.1 --save-exact  # для точной версии
```

## Полезные советы

- Использование точной версии (`--save-exact`) помогает избежать непредсказуемого поведения при обновлении зависимостей
- Для критичных пакетов рекомендуется всегда указывать точную версию
- В командной разработке желательно использовать `package-lock.json` или `yarn.lock` для обеспечения идентичности версий у всех разработчиков
