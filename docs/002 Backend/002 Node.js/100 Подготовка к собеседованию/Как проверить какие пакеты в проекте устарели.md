Чтобы проверить, какие пакеты в вашем проекте устарели, можно использовать несколько инструментов, включая встроенные возможности npm и сторонние утилиты.

### 1. **Использование команды `npm outdated`**

**`npm outdated`** — это встроенная команда, которая выводит список устаревших пакетов в вашем проекте, показывая текущие и последние доступные версии.

#### Пример использования:

bash

КопироватьРедактировать

`npm outdated`

### Что выводит команда:

- **`Package`** — название пакета.
- **`Current`** — текущая версия пакета, установленная в вашем проекте.
- **`Wanted`** — версия пакета, которую **npm** рекомендует для установки в соответствии с диапазоном версий в **`package.json`**.
- **`Latest`** — последняя доступная версия пакета в репозитории npm.
- **`Location`** — путь к пакету в **`node_modules`** (опционально).

#### Пример вывода:

bash

КопироватьРедактировать

`Package    Current  Wanted  Latest  Location lodash     4.17.15  4.17.21  4.18.0  node_modules/lodash express    4.17.1  4.17.1  5.0.0  node_modules/express`

### 2. **Использование `npm-check-updates` (ncu)**

Если вы хотите увидеть и обновить все устаревшие пакеты до самых последних версий, включая мажорные обновления, можно использовать сторонний инструмент **`npm-check-updates`** (ncu).

#### Шаги:

1. Установите **`npm-check-updates`**:
    
    bash
    
    КопироватьРедактировать
    
    `npm install -g npm-check-updates`
    
2. Просмотрите устаревшие пакеты:
    
    bash
    
    КопироватьРедактировать
    
    `npx npm-check-updates`
    
    Это покажет все устаревшие зависимости в вашем проекте и их последние доступные версии.
    
3. Чтобы обновить все зависимости до последних версий:
    
    bash
    
    КопироватьРедактировать
    
    `npx npm-check-updates -u`
    
4. Затем установите обновленные версии:
    
    bash
    
    КопироватьРедактировать
    
    `npm install`
    

### 3. **Использование `npm audit`**

Если вы хотите проверить также уязвимости и проблемы безопасности в устаревших пакетах, используйте команду **`npm audit`**.

bash

КопироватьРедактировать

`npm audit`

Это покажет информацию о безопасности и устаревших версиях с рекомендациями по их обновлению.

### Заключение:

- **`npm outdated`** — для проверки устаревших пакетов в проекте.
- **`npm-check-updates` (npx ncu)** — для обновления всех зависимостей до последних версий.
- **`npm audit`** — для проверки уязвимостей и устаревших версий с точки зрения безопасности.
