В Node.js REPL (Read-Eval-Print Loop) можно загрузить внешний модуль, например **lodash**, используя стандартную функцию `require()`. Это позволяет интерактивно использовать внешние библиотеки для тестирования или изучения их API.

## Шаги для загрузки внешнего модуля в REPL

### 1. Откройте REPL

Запустите Node.js REPL, выполнив команду в терминале:

```bash
node
```

Вы увидите приглашение командной строки `>`, означающее, что REPL готов принимать команды.

### 2. Загрузите внешний модуль

Используйте команду `require` для загрузки нужного модуля:

```javascript
const _ = require("lodash")
```

Здесь мы загружаем библиотеку lodash и присваиваем её переменной `_` (это стандартное соглашение для lodash).

### 3. Используйте функции загруженного модуля

После загрузки, функции модуля становятся доступными для использования:

```javascript
// Пример использования lodash
_.camelCase("hello world") // Возвращает: 'helloWorld'
_.chunk([1, 2, 3, 4, 5], 2) // Возвращает: [[1, 2], [3, 4], [5]]
_.capitalize("fred") // Возвращает: 'Fred'
```

## Предварительные условия и важные замечания

### Установка модуля

Перед использованием модуля в REPL, убедитесь, что он установлен в вашем проекте:

```bash
# Установка локально в текущем проекте
npm install lodash

# Или глобально (не рекомендуется для большинства модулей)
npm install -g lodash
```

### Область видимости модулей

REPL автоматически имеет доступ к:

- Всем стандартным модулям Node.js (например, `fs`, `http`, `path` и т.д.)
- Локально установленным модулям в директории `node_modules` текущего проекта
- Глобально установленным модулям (если требуется)

### Дополнительные возможности REPL

В REPL также можно:

```javascript
// Просмотреть доступные методы модуля
Object.keys(_)

// Получить справку по методу (если есть документация)
_.chunk // Посмотреть определение функции

// Использовать последний результат с помощью _
Math.PI // 3.141592653589793
Math.round(_) // 3
```

## Сохранение сессии REPL

Если вы часто используете определенные модули в REPL, можно создать файл `.repl_history` с предварительно загружаемыми модулями или настроить среду Node.js для автоматической загрузки модулей при запуске REPL.
