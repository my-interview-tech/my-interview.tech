Node.js обрабатывает большое количество одновременных соединений благодаря **однотредовому, событийно-ориентированному** циклу и **асинхронной** модели ввода-вывода (I/O).

Основные механизмы:
1. **Event Loop** – основной цикл событий, который управляет асинхронными операциями без блокировки основного потока.
2. **Non-blocking I/O** – операции чтения/записи выполняются асинхронно через `libuv`, освобождая поток для обработки других задач.
3. **Callbacks & Promises** – обеспечивают обработку завершения асинхронных операций.
4. **Worker Threads** – отдельные потоки для выполнения CPU-интенсивных задач.
5. **Cluster Mode** – позволяет масштабировать сервер, создавая несколько процессов Node.js, использующих все ядра процессора.
