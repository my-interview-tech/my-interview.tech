**npm (Node Package Manager)** — это стандартный менеджер пакетов для экосистемы Node.js, который позволяет разработчикам управлять зависимостями, устанавливать, обновлять и публиковать пакеты JavaScript.

### Основные функции npm:

1. **Управление зависимостями**:

   - Установка, обновление и удаление пакетов
   - Отслеживание версий в файле `package.json`
   - Автоматическое разрешение зависимостей пакетов

2. **Доступ к репозиторию пакетов**:

   - Поиск и загрузка пакетов из npm Registry (более 1.3 млн пакетов)
   - Использование как публичных, так и приватных пакетов

3. **Управление версиями**:

   - Семантическое версионирование (SemVer)
   - Фиксация версий в `package-lock.json` для воспроизводимых сборок

4. **Публикация и распространение пакетов**:

   - Возможность публиковать собственные пакеты
   - Управление доступом и видимостью пакетов

5. **Запуск скриптов и автоматизация**:
   - Определение скриптов в `package.json`
   - Автоматизация задач сборки, тестирования и развертывания

### Основные команды npm:

```bash
# Инициализация нового проекта
npm init

# Быстрая инициализация с дефолтными значениями
npm init -y

# Установка пакета локально
npm install express

# Установка пакета как зависимости разработки
npm install --save-dev jest

# Глобальная установка пакета
npm install -g typescript

# Установка конкретной версии пакета
npm install lodash@4.17.21

# Обновление пакетов
npm update

# Запуск скрипта
npm run start

# Публикация пакета
npm publish

# Поиск пакетов
npm search react
```

### Файлы npm:

1. **package.json** — основной файл конфигурации проекта, содержит:

   - Метаданные проекта (имя, версия, описание)
   - Зависимости и их версии
   - Скрипты для автоматизации задач
   - Настройки конфигурации

2. **package-lock.json** — фиксирует точные версии всех пакетов и их зависимостей для гарантии воспроизводимых сборок.

3. **node_modules/** — директория, где хранятся все установленные пакеты.

### Пример package.json:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "Пример Node.js приложения",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest"
  },
  "dependencies": {
    "express": "^4.18.2",
    "mongoose": "^6.9.0"
  },
  "devDependencies": {
    "jest": "^29.4.1",
    "nodemon": "^2.0.20"
  }
}
```

### Преимущества npm:

1. **Огромная экосистема** — доступ к миллионам готовых решений
2. **Простота использования** — понятный интерфейс и документация
3. **Гибкость** — поддержка различных конфигураций и версий
4. **Мощные инструменты для управления зависимостями**
5. **Интеграция с CI/CD-системами** для автоматизации сборки и публикации

npm стал неотъемлемой частью современной разработки на JavaScript и Node.js, значительно упрощая процесс управления зависимостями и расширяя возможности для повторного использования кода.
