Чтобы узнать, какие пакеты установлены в проекте, в **Node.js** можно использовать несколько подходов:

## 1. Просмотр файла `package.json`

В файле `package.json` содержится информация о зависимостях проекта, включая как **локальные**, так и **глобальные** пакеты. Этот файл можно открыть в любом текстовом редакторе и просмотреть секции зависимостей.

- **`dependencies`** — здесь указаны основные зависимости, которые необходимы для работы приложения.
- **`devDependencies`** — здесь указаны пакеты, необходимые только для разработки (например, тестовые фреймворки, инструменты для сборки и т. д.).

### Пример:

```json
{
  "dependencies": {
    "express": "^4.17.1",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^26.6.3"
  }
}
```

## 2. Использование команды `npm list`

Команда `npm list` показывает дерево зависимостей, установленных в вашем проекте. Это будет включать как прямые зависимости, так и их транзитивные (вложенные) зависимости.

### Для всех зависимостей проекта:

```bash
npm list
```

### Для только локальных зависимостей:

```bash
npm list --depth=0
```

Опция `--depth=0` ограничивает вывод только первичными зависимостями (без вложенных).

### Для глобальных зависимостей (установленных с флагом `-g`):

```bash
npm list -g --depth=0
```

### Пример вывода:

```
project@1.0.0 /path/to/your/project
├── express@4.17.1
├── lodash@4.17.21
└── jest@26.6.3
```

## 3. Просмотр папки `node_modules`

Каждый пакет, установленный с помощью `npm install`, будет находиться в папке **`node_modules`**. Вы можете просто открыть эту папку в файловом менеджере или с помощью командной строки, чтобы увидеть список установленных пакетов.

```bash
ls node_modules
```

Обратите внимание, что этот метод может показать больше пакетов, чем указано в `package.json`, так как включает все зависимости зависимостей.

## 4. Использование `npm outdated`

Команда `npm outdated` покажет, какие пакеты установлены в вашем проекте, а также доступные обновления для них. Она не только выводит установленные пакеты, но и проверяет, актуальны ли их версии.

```bash
npm outdated
```

### Пример вывода:

```
Package  Current  Wanted  Latest  Location
express    4.17.1  4.17.3  4.18.1  project
lodash    4.17.20  4.17.21  4.17.21  project
```

## 5. Использование `npm ls` для конкретного пакета

Если вы хотите узнать, установлен ли конкретный пакет в проекте, вы можете указать его имя:

```bash
npm ls <пакет>
```

### Пример:

```bash
npm ls lodash
```

## 6. Использование `npm why` для анализа зависимостей

Чтобы узнать, почему конкретный пакет установлен и кто его требует в качестве зависимости:

```bash
npm why <пакет>
```

### Пример:

```bash
npm why express
```

## Заключение

- **`package.json`**: Основной файл, где указываются все зависимости.
- **`npm list`**: Показывает дерево зависимостей.
- **`npm outdated`**: Проверяет доступные обновления для установленных пакетов.
- **`ls node_modules`**: Смотрит содержимое директории, где хранятся все пакеты проекта.
- **`npm why`**: Показывает, какие пакеты требуют конкретную зависимость.

Эти методы помогут вам узнать, какие пакеты установлены в вашем проекте и как их можно обновить или управлять ими.
