Переменная **`NODE_ENV`** используется в Node.js для определения текущей среды выполнения приложения (например, `development`, `production`, `test`).

**Основное назначение `NODE_ENV`**
1. Позволяет разделять конфигурации для разных сред (разработка, продакшен).
2. Помогает **оптимизировать производительность** в продакшене.
3. Используется многими библиотеками (например, Express, Webpack) для включения/выключения отладочных функций.

**Как получить доступ к переменным окружения?**

1. **Через `process.env`**  
    Node.js предоставляет глобальный объект `process.env`, который содержит все переменные окружения.

```js
console.log(process.env.NODE_ENV); // 'development' или 'production' 
console.log(process.env.PORT);     // Например, 3000`
```

2. **Задание переменных окружения:**

- В **Linux/macOS**:

```sh
export NODE_ENV=production node app.js
```

- В **Windows (cmd):**

```cmd
set NODE_ENV=production node app.js
```

3. **Использование `.env` файла (библиотека `dotenv`)**  
    В больших проектах удобно хранить переменные в `.env` файле:

```env
PORT=4000 NODE_ENV=development`
```

Затем подключаем через `dotenv`:

```js
require('dotenv').config(); console.log(process.env.PORT); // 4000`
```

**Когда это полезно?**
- Разделение **конфигураций для dev/prod**.
- **Хранение секретных ключей** (API-ключи, токены).
- Избежание хардкода в коде.

Лучше **не хранить чувствительные данные в коде** — используйте переменные окружения!
